WPR 281 Project Summery

This specification describes how to create an application that tracks the assignment, status, and progress of issues related to a project. The scope has been deliberately limiteddue to time constraints.I would however encourage you to add features and add thisprojectto your repositories on GitHub.


The system must be used via a web interface created using markup. Here is an example of an interface that can be used to build the system. It is just a sample,and you must only use it to visualise the kind of system that is required. There are plenty of other examples that you can look at for ideas.The picture displayed below only shows the summary of issues.


Once an issue is clicked, details of the issue must be displayedin an intuitive fashion. More on this later in the functional requirements.
